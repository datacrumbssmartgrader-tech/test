## 📝 Database Features

### QR Code System Ready
- Each table has a unique `qr_token` (UUID) auto-generated
- QR codes can be regenerated by admin to invalidate old ones
- Embedded in session URL: `https://app.com/dine?table=T01&token=<qr_token>`

### Customer Loyalty Tracking
- Customers matched by phone number
- Tracks: total_sessions, total_spent, first_visit, last_visit
- Enables returning customer identification

### Multi-Billing Support
- Sessions support multiple billing rounds (split bills)
- Payments link to specific orders via `order_ids[]` array
- Running total of payments per session

### Performance Optimized
- 16 indexes on frequently queried columns
- Foreign key constraints with cascade deletes
- Timestamptz for timezone-aware timestamps

## 📚 Schema Documentation

### staff table
Stores admin/user accounts with role-based access.
```
Columns: id (UUID), name (TEXT), role (enum: admin/user), 
         pin_hash (TEXT), avatar_initials (TEXT), active (BOOLEAN), created_at (TIMESTAMPTZ)
```

### customers table
Dine-in customer loyalty tracking.
```
Columns: id (UUID), name (TEXT), email (TEXT), phone (TEXT - UNIQUE),
         total_sessions (INT), total_spent (NUMERIC), first_visit (TIMESTAMPTZ),
         last_visit (TIMESTAMPTZ), created_at (TIMESTAMPTZ)
```

### restaurant_tables table
Dining tables with QR code management.
```
Columns: id (TEXT - PK, e.g. "T01"), label (TEXT), status (enum: empty/active/disabled),
         active_session_id (UUID FK), alert_active (BOOLEAN), qr_token (UUID - UNIQUE),
         qr_regenerated_at (TIMESTAMPTZ), created_at (TIMESTAMPTZ)
```

### sessions table
One visit = one session, tied to customer and table.
```
Columns: id (UUID), table_id (TEXT FK), customer_id (UUID FK),
         billing_round (INT), opened_at (TIMESTAMPTZ), closed_at (TIMESTAMPTZ),
         total_paid (NUMERIC)
```

### menu_items table
Restaurant menu catalog.
```
Columns: id (UUID), name (TEXT), description (TEXT), category (TEXT),
         price (NUMERIC), prep_time (INT), image_url (TEXT), image_public_id (TEXT),
         type (enum: single/platter), available (BOOLEAN), hidden (BOOLEAN),
         discount_type (enum: none/percent/flat), discount_value (NUMERIC),
         created_at (TIMESTAMPTZ), updated_at (TIMESTAMPTZ)
```

### orders table
Customer order batches.
```
Columns: id (UUID), session_id (UUID FK), table_id (TEXT FK),
         billing_round (INT), status (enum: received/kitchen/ready/served/cancelled),
         cancel_reason (TEXT), total (NUMERIC), placed_at (TIMESTAMPTZ),
         updated_at (TIMESTAMPTZ)
```

### order_items table
Individual line items in orders.
```
Columns: id (UUID), order_id (UUID FK - CASCADE), menu_item_id (UUID FK),
         name (TEXT), price (NUMERIC), qty (INT), note (TEXT), created_at (TIMESTAMPTZ)
```

### payments table
Payment records linking to orders.
```
Columns: id (UUID), session_id (UUID FK), table_id (TEXT FK),
         billing_round (INT), order_ids (UUID[]), amount (NUMERIC),
         method (enum: card/cash), paid_at (TIMESTAMPTZ)
```

### waiter_alerts table
Waiter call, bill request, and complaint tracking.
```
Columns: id (UUID), table_id (TEXT FK), session_id (UUID FK),
         type (enum: waiter/bill/complaint), message (TEXT), dismissed (BOOLEAN),
         created_at (TIMESTAMPTZ)
```

### menu_extras table
Add-on options for menu items.
```
Columns: id (UUID), menu_item_id (UUID FK - CASCADE), label (TEXT),
         price (NUMERIC), created_at (TIMESTAMPTZ)
```

## 📞 Troubleshooting

If you need to reset the database:
1. Go to [console.neon.tech](https://console.neon.tech)
2. Navigate to your project "Roosters Den"
3. In Settings → Databases, delete and recreate the database
4. Re-run the seed script: `npm run seed`

If connection fails:
1. Verify `DATABASE_URL` in `.env.local` is correct
2. Check Neon console for project status
3. Ensure IP whitelisting is not blocking your connection
